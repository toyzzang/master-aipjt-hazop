# HAZOP 위험성평가 AI 의미 검토 절차

시스템 코드가 이미 확인한 Node/변수/Guideword 일치, Row 개수, 숫자 범위와 위험도 계산을 다시 대신하지 않는다.

다음을 의미 관점에서 검토하고, 문제가 있으면 전체 Row의 문장·빈도·강도 후보와 근거를 보완한다.

1. 원인이 실제 설비와 운전에서 현실적인가?
2. 결과가 작성된 원인과 자연스럽게 연결되는가?
3. Silane 등 고위험 물질 누출의 강도를 지나치게 낮게 평가하지 않았는가?
4. 현재 안전조치가 원인 예방 조치인지 결과 완화 조치인지 구분되며, 핵심 방어가 빠지지 않았는가?
5. MSDS의 유해성·반응성·누출 대응 정보와 결과 문장이 모순되지 않는가?
6. 사고이력 또는 표준 HAZOP보다 위험을 낮게 평가했다면 충분한 근거가 있는가?
7. 자료가 부족한 판단에 `확인 필요`가 명시됐는가?

Node, 변수, Guideword, Row 개수는 절대 바꾸지 않는다. 검토 결과는 지적 목록만 반환하지 말고 보완한 전체 위험성평가 Row에 반영한다.
