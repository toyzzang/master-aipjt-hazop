# Branch Workflow Rules Worklog

## Goal

- 앞으로 소스 변경이 발생하는 작업은 `main`에서 직접 하지 않고 별도 브랜치에서 진행한다.
- 기능 개발은 `feature/...` 브랜치에서 진행한다.
- 버그 수정은 `bug/...` 브랜치에서 진행한다.

## Scope

- 프로젝트 작업 규칙 문서에 브랜치 운영 규칙을 추가한다.
- Codex 개발 하네스의 수정 전 절차에도 브랜치 생성 원칙을 반영한다.

## Success Criteria

- `AGENTS.md`에 feature/bug 브랜치 규칙이 명시된다.
- `note/codex-development-harness.md`에 소스 수정 전 브랜치 생성 절차가 명시된다.
- 이번 문서 변경 작업은 `feature/branch-workflow-rules` 브랜치에서 수행한다.
