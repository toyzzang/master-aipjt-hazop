# 진행 문구 및 최종 검증 로그 정리

## 문제

- #3/#4 결과 영역이 완료된 과거 Agent 로그까지 검색해 `독립 의미 검토 진행 중`을 계속 표시한다.
- DeepAgents 내부 구현인 `read_file` 성공 로그가 사용자 화면에 노출된다.
- 전체 Agent Workflow가 끝난 뒤 서비스 계층이 모든 위험성평가 Row를 다시 로그로 출력하여, 위험도 계산이 조치계획 Agent 뒤에 실행되는 것처럼 보인다.

## 변경

- 결과 영역은 `loading=true`인 현재 Agent 부모 블록만 기준으로 진행 문구를 표시한다.
- 현재 Agent가 없으면 가장 최근 시스템/검증/결과 이벤트를 기준으로 안내한다.
- Skill `read_file` trace는 필수 Skill 실행 검증에는 사용하되 화면 로그로 내보내지 않는다.
- Agent 완료 뒤 Row별 위험도 로그를 제거하고 최종 검증 요약 1건만 표시한다.

## 실제 처리 순서

1. Excel 검증 및 위험도 기준표 파싱
2. 입력 물질별 최초 MSDS 1회 조회
3. risk-draft-agent 초안 작성
4. 시스템 1차 검증 및 위험도 계산
5. risk-review-agent 의미 검토 및 초안 반영
6. 시스템 재검증, 위험도 재계산, 고위험 대상 선별
7. action-plan-agent 조치계획 작성
8. 조치계획 시스템 검증 및 잔여 위험도 계산
9. 결과 요약 로그 및 Excel 저장

## 성공 기준

- 완료된 검토 Agent를 결과 영역에서 계속 `진행 중`으로 표시하지 않는다.
- 화면 로그에 `read_file`이 나타나지 않는다.
- Agent 종료 후 위험도 Row별 로그가 반복되지 않는다.
- 전체 테스트, Frontend typecheck/build가 통과한다.
