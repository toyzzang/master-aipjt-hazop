# 작업 설계: Streamlit 2분할 화면과 Agent 로그 개선

## 배경

현재 PoC 화면은 FastAPI 정적 HTML/JS로 구성되어 있다.
사용자는 입력 영역과 Agent 로그 영역을 좌우로 나누어 보고 싶어 하며, 로그가 너무 빠르게 지나가므로 진행 중임을 보여주는 로딩 애니메이션과 단계별 문구가 필요하다.

## 목표

1. Streamlit 기반 화면을 추가한다.
2. 왼쪽은 Excel/기본정보 입력, 오른쪽은 Agent 로그 표시 영역으로 구성한다.
3. 로그는 한 번에 몰리지 않고 하나씩 읽히도록 표시한다.
4. LLM 대기 문구는 단순 "응답 대기"가 아니라 원인/결과/안전조치/조치계획 생성 단계처럼 보이게 한다.
5. Bing 일반 웹 검색 fallback은 일단 제외한다.
6. "환경 변수가 설정되어 있어 실제 LLM" 문구는 제거하고, 연결된 모델을 이용한다는 표현으로 바꾼다.

## 구현 방침

- 기존 `run_hazop_agent` 흐름은 재사용한다.
- Streamlit 앱은 별도 진입점 `app/streamlit_app.py`로 추가한다.
- 결과 다운로드는 Streamlit의 `download_button`으로 제공한다.
- Agent 내부 로그 문구는 사용자에게 보이는 표현만 수정한다.
- Bing 검색은 코드상 임시 비활성화하고, KOSHA 실패 시 내장 요약으로 fallback한다.

## 성공 기준

- `streamlit run app/streamlit_app.py`로 좌우 2분할 화면이 뜬다.
- Excel 업로드 후 Agent 로그가 오른쪽에 단계별로 표시된다.
- 진행 중인 최신 로그에는 동그라미 로딩 애니메이션이 보인다.
- LLM 대기 로그가 "원인을 생성중입니다", "결과를 생성중입니다" 같은 단계 문구로 표시된다.
- Bing 검색 로그가 나타나지 않는다.
- Python compile 검증을 통과한다.
