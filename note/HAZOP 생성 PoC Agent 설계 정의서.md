## 1. 개요

**2등급 신규 Maker/Model 도입 시 공정위험성평가(HAZOP) 초안을 생성하는 AI Agent**의 프로세스 정의서와 기능 정의서를 정리하기 위한 초안입니다.

핵심 역할은 **사용자가 작성한 기존 HAZOP 양식의 `#1 노드리스트`, `#2 가이드워드` Sheet와 기본정보, MSDS 기준정보, 표준공정위험성평가서 Link를 기반으로 `#3 위험성평가`, `#4 조치계획서` 초안을 생성하는 것**입니다.

이번 PoC에서는 AI가 노드별 Guideword/Parameter 후보를 추천하지 않습니다. Guideword/Parameter는 사용자가 `#2 가이드워드` Sheet에 작성해서 업로드하는 것으로 전제합니다.

## 1-1. 목적

| 구분        | 내용                                               |
| --------- | ------------------------------------------------ |
| Agent 목적  | 신규 Maker/Model 도입 시 HAZOP 초안 작성 업무를 보조           |
| 핵심 역할     | 원인, 결과 또는 영향, 현재 안전조치, 빈도/강도 후보, 개선조치 초안 생성      |
| 최종 목표     | 작성 시간 단축, 표준 HAZOP 문서 재활용, DB Insert 자동화         |
| 제외 사항     | 노드 자동 생성, Guideword/Parameter 자동 추천, HAZOP 최종 승인 |
| 최종 Output | Hazop 문서의 #3,#4 sheet를 자동화 완성                    |

## 2. 전체 To-Be 프로세스

```
[1] 사용자가 HAZOP 등록화면에서 [AI 초안생성] 버튼 클릭
        ↓
[2] 시스템이 AI 초안생성 입력 창 표시
        ↓
[3] 사용자가 기존 HAZOP 양식 Excel 업로드
    - #1 노드리스트 Sheet 작성 완료
    - #2 가이드워드 Sheet 작성 완료
        ↓
[4] 사용자가 기본정보 및 참고문서 정보 입력
    - Maker/Model
    - MSDS기반 주요 물질정보 (만일, Node별 물질 정보가 다를 경우 Node별 물질 입력)
    - 유사사례로 참고할 표준공정위험성평가서 문서 Link
        ↓
[5] 시스템이 업로드 Excel과 입력값 검증
        ↓
[6] 시스템이 Excel의 #1 노드리스트, #2 가이드워드 Sheet 파싱
        ↓
[7] 시스템이 MSDS 기준정보와 표준공정위험성평가서 문서 정보 조회
        ↓
[8] AI Agent가 노드/가이드워드/MSDS/표준공정위험성평가서 문서를 분석
        ↓
[9] AI Agent가 #3 위험성평가 Sheet 초안 생성
    - 원인
    - 결과 또는 영향
    - 현재 안전조치
    - 빈도 후보
    - 강도 후보
        ↓
[10] 시스템이 기준표에 따라 위험도 계산
        ↓
[11] AI Agent가 위험도 기준 이상 항목에 대해 #4 조치계획서 Sheet 초안 생성
    - 개선조치
    - 조치 후 빈도 후보
    - 조치 후 강도 후보
        ↓
[12] 시스템이 생성 결과를 DB Insert
        ↓
[13] 시스템이 생성 결과를 화면에 표시
        ↓
[14] 사용자가 생성 결과 검토 및 수정
        ↓
[15] 최종 확정
        ↓
[16] 확정 문서 다운로드 및 이력 관리
```

## 4. 질의사항
1. 2등급은 신규 Maker/Model인데 유사사례로 이용할만한 Hazop 문서들이 있으면 좋을것같다. But, 어떤 기준으로 유사하다고 판단 할 수 있을까?
2. MSDS 스키마 테이블/컬럼 정보
3. Node별 물질정보가 다른 경우가 있는지?
4. 빈도, 강도 산정하기 위한 참고할 데이터는 어떤 것들이 있을까?
5. 사고이력, 정비이력, 이상발생 이력등을 현재 관리하고 있는지? (빈도 산정에 이용하기 위함)
   없다면.. 유사설비에 대한 빈도 데이터를 봐야하는데, 유사설비에 대한 판단 기준은 무엇?
6. Hazop이 모델별로 생성하는건데 장비정보(eq id)도 관리를 하고있는것인지? 
: 인허가 ID 하나에 여러가지 EQID(장비)가 존재함. EQID를 기준으로 각각 문서 75종을 생성한다. eqid:문서=1:1
사고이력에 대한 설비는 표준공정위험성평가에 Link해둔 

Eq Master id = 각 장비별 고유한 갑사. MAC주소 개념
Eq id = 설치 위치에 따라 바뀌는 정보. IP주소 개념

공정별로 노드가 1:N임


## 5. 필요사항

1. Hazop문서는 Maker/Model별로 생성되는 것이 맞는지?
2. 신규 Maker/Model기반 문서 생성시 기존 유사한 다른 Maker/Model 문서 참조할때, 유사하다고 판단할 수 있는 기준은 무엇인지? 
3. Node별 물질 정보가 다른 경우가 있는지? > 1개의 노드당 여러 물질사용 가능하며, 노드별로 물질정보가 다르다.
4. 표준공정위험성평가 문서에서 Eqid를 관리하고 있는지? (Eqid가 존재해야 유사설비로 보고 대한 빈도데이터 참조 가능)
5. MSDS 스키마 정보 필요
6. 사고이력,정비이력,이상발생 이력 등 장비에 대한 이력 관리 스키마 정보 필요
