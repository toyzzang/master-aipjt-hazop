# HAZOP 생성 PoC 시나리오 정의서 v2

## 1. 개요

본 문서는 변경된 HAZOP 생성 PoC 프로세스를 기준으로 작성한 시나리오 정의서입니다.

기존 시나리오는 사용자가 장비, 공정 설명, 정상운전 조건 등을 직접 입력하면 AI Agent가 HAZOP 검토결과를 생성하는 흐름이었습니다.

변경된 프로세스에서는 사용자가 이미 작성한 기존 HAZOP Excel 양식을 업로드합니다. 업로드 Excel에는 `#1 노드리스트` Sheet와 `#2 가이드워드` Sheet가 작성되어 있어야 합니다.

AI Agent는 사용자가 업로드한 `#1 노드리스트`, `#2 가이드워드`, Maker/Model, MSDS 기준정보, Node별 물질정보, 표준공정위험성평가서 Link를 참고하여 `#3 위험성평가` Sheet와, 필요한 경우 `#4 조치계획서` Sheet 초안을 생성합니다.

이번 PoC에서는 AI가 Node, Guideword, Parameter를 자동 추천하지 않습니다. Node는 `#1 노드리스트` Sheet에, Guideword/Parameter는 `#2 가이드워드` Sheet에 사용자가 미리 작성한다고 가정합니다.

## 2. 공통 전제

- 사용자는 HAZOP 등록화면에서 `[AI 초안생성]` 버튼을 클릭할 수 있어야 합니다.
- 사용자는 기존 HAZOP 양식 Excel을 업로드할 수 있어야 합니다.
- 업로드 Excel에는 `#1 노드리스트` Sheet가 작성되어 있어야 합니다.
- 업로드 Excel에는 `#2 가이드워드` Sheet가 작성되어 있어야 합니다.
- 사용자는 Maker/Model 정보를 입력할 수 있어야 합니다.
- 사용자는 MSDS 기반 주요 물질정보를 입력할 수 있어야 합니다.
- Node별 물질정보가 다른 경우, Node별 물질정보를 입력할 수 있어야 합니다.
- 사용자는 유사사례로 참고할 표준공정위험성평가서 Link를 입력할 수 있어야 합니다.
- 사내 위험도 기준표는 시스템에 미리 등록되어 있다고 가정합니다.
- 시스템은 기준표에 따라 조치 전 위험도와 조치 후 위험도를 계산합니다.
- AI Agent가 생성한 결과는 최종 승인 문서가 아니라 담당자 검토용 초안입니다.

## 3. 시나리오 1: 위험도가 낮아 추가 조치가 필요 없는 HAZOP 초안 생성

### 3.1 기본 정보

| 항목 | 내용 |
| --- | --- |
| ID | SC-001 |
| 시나리오명 | 위험도가 낮아 추가 조치가 필요 없는 HAZOP 초안 생성 |
| 목적 | 업로드한 HAZOP Excel과 참고정보를 기반으로 `#3 위험성평가` 초안을 생성하고, 위험도가 낮은 항목은 `#4 조치계획서`를 생성하지 않고 현 관리 유지로 표시 |

### 3.2 상황

사용자가 신규 Maker/Model 장비에 대한 HAZOP 초안을 생성하기 위해 HAZOP Excel 양식을 업로드합니다.

업로드한 Excel에는 `#1 노드리스트` Sheet와 `#2 가이드워드` Sheet가 이미 작성되어 있습니다. 사용자는 추가로 Maker/Model, 주요 물질정보, Node별 물질정보, 표준공정위험성평가서 Link를 입력합니다.

AI Agent는 해당 정보를 기반으로 `#3 위험성평가` Sheet 초안을 생성합니다. 시스템이 위험도를 계산한 결과, 모든 항목이 사내 기준상 추가 조치가 필요 없는 수준으로 평가됩니다.

### 3.3 사전 조건

- 사용자는 기존 HAZOP 양식 Excel을 업로드할 수 있어야 합니다.
- 업로드 Excel에는 `#1 노드리스트` Sheet가 작성되어 있어야 합니다.
- 업로드 Excel에는 `#2 가이드워드` Sheet가 작성되어 있어야 합니다.
- AI는 Node, Guideword, Parameter를 자동 추천하지 않습니다.
- 사용자는 Maker/Model 정보를 입력할 수 있어야 합니다.
- 사용자는 MSDS 기반 주요 물질정보를 입력할 수 있어야 합니다.
- Node별 물질정보가 다른 경우, Node별 물질정보를 입력할 수 있어야 합니다.
- 사용자는 유사사례로 참고할 표준공정위험성평가서 Link를 입력할 수 있어야 합니다.
- 사내 위험도 기준표는 시스템에 미리 등록되어 있습니다.
- 위험도가 사내 기준상 낮거나 허용 가능한 경우, 별도 권고 조치를 생성하지 않습니다.
- AI가 생성한 결과는 최종 승인 문서가 아니라 담당자 검토용 초안입니다.

### 4.8 성공 기준

- 업로드한 Excel의 `#1 노드리스트`, `#2 가이드워드`를 기반으로 `#3 위험성평가` 초안이 생성되어야 합니다.
- AI는 Node와 Guideword/Parameter를 새로 추천하지 않아야 합니다.
- 시스템은 사내 기준표에 따라 조치 전 위험도를 계산해야 합니다.
- 위험도가 기준 이상인 항목만 `#4 조치계획서` 생성 대상이 되어야 합니다.
- `#4 조치계획서`에는 개선조치, 조치 후 강도, 조치 후 빈도, 조치 후 위험도가 포함되어야 합니다.
- 생성 결과는 DB에 저장되고 화면에 표시되어야 합니다.
