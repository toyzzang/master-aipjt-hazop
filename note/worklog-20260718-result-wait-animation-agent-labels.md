# 작업 설계: 결과 대기 애니메이션과 Agent 역할 한글 안내

## 배경

`#3 위험성평가 / #4 조치계획서` 결과 영역이 처리 중일 때 회전형 스피너가 표시된다.
사용자는 이 영역에서는 급하게 도는 느낌보다, 결과를 차분히 기다리는 느린 페이드 인·아웃 효과를 원한다.
또한 실시간 로그의 `risk-draft-agent` 같은 내부 이름만으로는 각 Agent의 역할을 이해하기 어렵다.

## 변경 범위

1. 결과 영역의 회전형 스피너를 제거한다.
2. 느리게 밝아졌다 어두워지는 3개 표시점으로 대기 상태를 표현한다.
3. 화면 읽기 도구가 처리 상태 문구를 읽을 수 있도록 상태 영역 접근성을 유지한다.
4. 실시간 로그의 전문 Agent 이름 뒤에 한글 역할을 덧붙인다.
   - `risk-draft-agent (위험성평가 초안 작성)`
   - `risk-review-agent (위험도 계산·근거 검토)`
   - `action-plan-agent (고위험 항목 조치계획 작성)`
5. 서버 원본 로그는 변경하지 않고 화면 표시 단계에서만 한글 설명을 추가한다.

## 성공 기준

- 결과 대기 영역에 회전형 스피너가 표시되지 않는다.
- 대기 표시는 약 3초 주기로 천천히 페이드 인·아웃한다.
- Agent 이름이 제목과 상세 로그 어디에 나타나든 한글 역할이 함께 표시된다.
- 이미 역할이 붙은 문자열에는 설명이 중복으로 추가되지 않는다.
- frontend typecheck와 production build가 성공한다.
- 기존 Python 테스트가 계속 통과한다.
