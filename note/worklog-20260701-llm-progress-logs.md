# 작업 설계: Azure OpenAI 긴 대기 구간 진행 로그 개선

## 배경

`Azure OpenAI Agent 생성을 시작합니다.` 이후 실제 LLM 호출이 오래 걸리면 화면에서 진행이 멈춘 것처럼 보인다.
사용자는 이 구간에서 Agent가 무엇을 하고 있는지 더 잘게 보고 싶어 한다.

## 설계 판단

LLM API 호출 자체는 하나의 네트워크 요청이라 내부 토큰 생성 과정을 서버가 실시간으로 알 수 없다.
하지만 호출 전후의 준비/검증 단계와 호출 중 heartbeat는 표시할 수 있다.

이번 PoC에서는 다음 로그를 추가한다.

1. `#3 위험성평가` 생성 대상 Row 수 확인
2. LLM 입력 Context 구성
3. Azure OpenAI 요청 전송
4. 응답 대기 heartbeat
5. 응답 JSON 수신
6. Row 배열 추출
7. Pydantic Schema 검증
8. 이후 위험도 계산

`#4 조치계획서` 생성도 동일하게 진행 로그를 추가한다.

## 수정 범위

- `app/services/llm.py`
  - 긴 LLM 호출 중 heartbeat 이벤트를 받을 수 있는 helper 추가
- `app/services/agent.py`
  - LLM 호출 전후 진행 로그 추가
  - `#3`, `#4` 각각 진행 단계 표시

## 비수정 범위

- LLM 프롬프트의 업무 내용은 변경하지 않는다.
- 위험도 계산 기준은 변경하지 않는다.
- DB 스키마는 변경하지 않는다.

## 성공 기준

- Azure OpenAI 호출 중 2초 이상 침묵하지 않는다.
- `#3` 생성과 `#4` 생성의 준비/요청/응답/검증 단계가 로그로 보인다.
- 샘플 3종 demo 흐름은 계속 통과한다.

## 구현 결과

- `#3 위험성평가` LLM 호출 전후 로그를 추가했다.
- `#4 조치계획서` LLM 호출 전후 로그를 추가했다.
- LLM 응답 대기 중 `AGENT_LLM_HEARTBEAT_SECONDS` 간격으로 heartbeat 로그를 표시한다.
