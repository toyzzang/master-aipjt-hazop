# Agent invoke 로그 복원 및 명칭 정리

## 목적

- DeepAgents `astream()` 기반 실제 실행 스트림을 제거한다.
- 직전 방식인 `agent.invoke()` 완료 대기와 사용자 설명용 단계 로그 방식으로 복원한다.
- 화면과 로그에서 `Sub Agent`, `독립 DeepAgent` 표현을 제거하고 모두 `Agent`로 표시한다.
- Agent 시작 설명은 내부 제어 방식보다 사용자가 이해할 작업 내용 중심으로 표시한다.
  - 예: `#1 노드리스트와 #2 가이드워드를 기반으로 #3 위험성평가 초안을 작성합니다.`

## 유지 사항

- Python Workflow가 작성 → 시스템 검증 → 검토 → 고위험 선별 → 조치계획 순서를 강제한다.
- Azure OpenAI는 Agent가 아니라 각 DeepAgent 안에 연결된 LLM 모델로 설명한다.
- 동일한 모델 대기 문구는 Agent별 한 번만 표시한다.
- 실행 완료 후 실제 Skill 읽기 여부와 실제 보완 조회 Tool trace를 요약 표시한다.
- 실패 시 Agent와 실패 단계 및 원인을 표시한다.

## 성공 기준

- 소스에 `agent.astream()` 호출이 남지 않는다.
- 세 전문 단계가 각각 `Agent` 상위 로그 블록으로 표시된다.
- 기존 Skill → Tool → Context → 모델 대기 설명 순서가 복원된다.
- Python 테스트, TypeScript 검사, 프런트엔드 빌드가 통과한다.

## 결과표 판단 근거 컬럼 색상

- #3 위험성평가의 판단·강도·빈도 근거 컬럼과 #4 조치계획서의 조치 판단 근거 컬럼에 연보라색 테마를 적용한다.
- 판단 근거 컬럼은 기존 헤더보다 조금 진한 회색을 헤더에만 사용하고 데이터 셀은 기존 흰색을 유지한다.
