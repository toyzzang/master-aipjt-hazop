# 결과 그리드 컬럼 한글화 및 검토 내역 명칭 변경

## 문제

- Frontend 컬럼 변환표에 `cause`, `existing_safeguard`, `decision_evidence` 등 실제 Pydantic 스키마의 단수형 필드가 빠져 영문으로 표시된다.
- `독립 AI 검토 의견`이라는 제목이 현재 `risk-review-agent(초안 검토 및 보완)` 역할명과 일치하지 않는다.

## 변경

- RiskAssessmentRow, ActionPlanRow, ReviewFinding의 모든 필드를 한글 컬럼명으로 매핑한다.
- `Guideword`, `Node`, `No.`처럼 일부 남아 있던 영문 표기도 한글화한다.
- 검토 결과 표 제목을 `초안 검토 및 보완 내역`으로 변경한다.
- 검토 결과가 없을 때는 `초안 검토 과정에서 별도 보완 내역이 없습니다.`라고 표시한다.

## 검토 내역 표를 유지하는 이유

- 검토 Agent가 어떤 문제를 발견했는지 기록한다.
- 강도, 원인, 결과, 근거를 어떻게 보완했는지 기록한다.
- 담당자 추가 확인이 필요한 항목을 구분한다.
- 검토 결과가 최종 위험성평가에 반영됐는지 추적할 수 있게 한다.

## 성공 기준

- #3, #4, 검토 내역 그리드 헤더에 스키마 필드명이 영문으로 표시되지 않는다.
- 검토 표 제목이 현재 Agent 역할과 일치한다.
- Frontend typecheck/build가 통과한다.
