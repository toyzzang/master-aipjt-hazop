# 작업 설계: 다양한 복합 HAZOP 샘플과 KOSHA 조회 확인

## 배경

현재 PoC는 반도체 설비 중심의 Excel 샘플 3개와 소수 물질용 내장 MSDS 요약에 집중되어 있다.
따라서 사용자가 여러 산업·여러 Node·여러 Guideword 조합을 시험하기 어렵고,
KOSHA 실검색이 성공했는지 또는 왜 내장 요약으로 전환됐는지 확인하기도 어렵다.

## 작업 범위

1. 기존 3개 샘플은 호환성을 위해 유지한다.
2. 서로 다른 산업과 위험 유형을 다루는 샘플을 추가한다.
   - 암모니아 냉동설비
   - IPA 용제 공급/회수설비
   - 염소 수처리설비
   - 배터리 전해액 혼합설비
   - 고압 증기 보일러
3. 최소 1개는 Node와 Guideword 수가 많은 복합 공정 샘플로 구성한다.
4. 샘플 정의를 한 곳에서 관리해 Excel 생성과 테스트 시나리오가 쉽게 늘어나도록 정리한다.
5. KOSHA 조회 결과에 성공/실패 원인을 담아 화면 로그에서 구분할 수 있게 한다.
6. KOSHA HTML 파싱과 정확 일치 선택을 네트워크 없이도 자동 테스트한다.
7. 가능하면 실제 KOSHA 사이트를 대상으로 대표 물질 조회를 별도로 실행해 현재 연결 상태를 확인한다.
8. README에 새 샘플의 목적과 KOSHA 확인 방법을 설명한다.

## 구현 원칙

- Excel의 `#1 노드리스트`, `#2 가이드워드`만 채운다.
- `#3 위험성평가`, `#4 조치계획서`는 생성 시 헤더만 둔다.
- AI/규칙 엔진은 Excel에 없는 Node, 변수, Guideword를 만들지 않는다.
- 같은 Node/변수의 여러 Guideword는 각각 별도 Row로 둔다.
- 위험도는 시스템이 `빈도 * 강도`로 계산한다.
- KOSHA 실사이트에 의존하는 테스트는 기본 테스트와 분리해 외부 장애로 CI가 깨지지 않게 한다.

## 성공 기준

- 총 8개 이상의 샘플 Excel이 생성된다.
- 신규 복합 샘플은 8개 이상의 Node와 20개 이상의 Guideword 조합을 가진다.
- 모든 샘플의 `#3`, `#4` 데이터 영역이 생성 직후 비어 있다.
- 저장된 전체 시나리오에서 위험성평가 Row 수가 Guideword 수와 정확히 같다.
- 위험도 9 이상 Row에 대해서만 조치계획서가 생성된다.
- KOSHA 응답 파서가 정확 일치 물질을 선택하고 짧은 검색어 오매칭을 거부한다.
- 실제 KOSHA 연결 검증 결과를 작업 기록과 최종 보고에 남긴다.
- `python -m compileall app scripts tests`와 전체 테스트가 통과한다.

## 범위 밖

- KOSHA 데이터를 로컬 DB에 대량 수집하는 기능
- 외부 검색 API(Bing 등) 재도입
- 실제 사업장 위험도 확정 또는 법적 판단 자동화

## 검증 결과

- 샘플 Excel 8개 생성 성공
  - 기존 기본 샘플 3개 유지
  - 신규 산업/복합 샘플 5개 추가
- 대형 통합 샘플: Node 10개, Guideword 평가 조합 33개
- 전체 샘플의 `#3`, `#4`가 헤더 1행만 가진 빈 Sheet임을 자동 테스트로 확인
- 전체 샘플을 demo fallback Engine으로 실행해 Excel의 평가 조합과 생성 위험성평가 Row가 정확히 일치함을 확인
- 위험도 9 이상 Row와 조치계획서 참조 Row가 정확히 일치함을 확인
- 전체 테스트: `14 passed`
- compileall 성공

### 실제 KOSHA 연결 확인 (2026-07-18)

외부망을 허용한 환경에서 현재 KOSHA 사이트를 직접 조회했다.

| 검색어 | 결과 | KOSHA chem_id |
|---|---|---|
| Hydrogen | 성공 | `000557` |
| Ammonia | 성공 | `001174` |
| Chlorine | 성공 | `001020` |
| Isopropyl alcohol | 성공 | `001065` |

각 상세 페이지에서 H문구/유해성 분류와 P문구/취급 문구를 추출했다.
샌드박스처럼 DNS가 차단된 환경에서는 KOSHA 조회가 실패하며, 이 경우 새 로그에
`ConnectError` 같은 실패 종류가 나타난 뒤 PoC 내장 MSDS 요약으로 전환된다.
