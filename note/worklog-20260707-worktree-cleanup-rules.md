# Worktree 정리 규칙 추가

## 배경

사용자는 앞으로 작업 브랜치를 `.worktrees/작업명` 아래에서 진행하기로 했다.
또한 PR을 올린 뒤 "정리해줘"라고 요청하면 Codex가 안전하게 로컬 작업 환경을 정리하길 원했다.

## 구현 범위

- `AGENTS.md`에 PR 이후 정리 규칙을 추가한다.
- `note/codex-development-harness.md`에 "정리해줘" 요청 시 확인/삭제 절차를 추가한다.
- 실제 worktree 삭제나 브랜치 삭제는 이번 문서 작업 범위가 아니다.

## 성공 기준

- push/PR 생성 여부를 확인하기 전에는 worktree와 로컬 브랜치를 삭제하지 않는다고 명시된다.
- PR이 확인된 경우 worktree 삭제와 로컬 브랜치 삭제 절차가 명시된다.
- 원격 브랜치는 PR 상태와 사용자 요청에 따라 별도로 판단한다고 명시된다.
