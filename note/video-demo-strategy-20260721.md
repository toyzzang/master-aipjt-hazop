# HAZOP AI Agent 시연 영상 구성 전략

## 영상 목표

5분 이내에 단순 결과 화면이 아니라 다음 기술적 흐름을 증명한다.

1. 입력 범위를 시스템이 고정한다.
2. Agent가 5단계 안팎의 작업 계획을 수립한다.
3. 필요한 경우에만 Tool을 호출하고 출처를 남긴다.
4. 별도 Review Agent가 초안을 검토하고 실제로 수정한다.
5. 위험도와 고위험 분기는 시스템 코드가 검증한다.

## 시연 시나리오

- 샘플: `HAZOP_ColdChain_NH3-Refrigeration.xlsx`
- 물질: Ammonia
- 강조할 입력: 최근 High Alarm, 감지기 경보, Mechanical Seal/Receiver 연결부 점검 이력
- 강조 근거: 암모니아 MSDS, 사용자 사고·정비 이력, 로컬 표준 HAZOP

## 5분 타임라인

### 0:00~0:25 · 문제와 시나리오 소개

화면:

- 업로드 전 Excel의 `#1 노드리스트`, `#2 가이드워드`
- 비어 있는 `#3 위험성평가`, `#4 조치계획서`

해설:

> 암모니아 냉동설비의 입력 Node와 Guideword는 바꾸지 않고 위험성평가와 고위험 조치계획 초안을 생성합니다. AI는 점수 후보와 근거를 만들고, 위험도 계산은 시스템 코드가 수행합니다.

### 0:25~0:55 · 업로드와 실행 시작

화면:

- Excel 업로드
- Node/Guideword 확인
- 운전 의도와 사고·정비 이력
- `AI 초안생성` 클릭

해설:

> 업로드된 Node·변수·Guideword 조합만 평가 대상으로 등록합니다.

### 0:55~1:35 · Agent Planning

화면:

- `로그 확대` 실행
- risk-draft-agent Planning
- 한국어 5단계 실행 현황

해설:

> 안전 Workflow 순서는 고정되어 있습니다. 각 Agent는 현재 작업을 한국어 체크리스트로 계획하고, 필요한 근거가 부족할 때만 Tool을 호출합니다.

주의:

- Planning 실행 현황과 완료 상태에서 2~3초 정지하거나 확대한다.

### 1:35~2:25 · Skill과 Tool Call

화면:

- 적용 Skill
- Tool 호출 목적
- Tool 입력
- KOSHA/MSDS 결과 출처
- Tool 결과가 전달되는 판단 영역

해설:

> Tool은 무조건 호출하지 않습니다. 최초 MSDS 요약만으로 유해성이나 대응 정보가 부족할 때 상세 조회합니다. KOSHA 검색 결과의 chem_id와 H/P 문구를 내부 요약 구조로 변환해 강도 및 안전조치 판단 입력으로 전달합니다.

편집:

- 순수 모델 대기 구간은 제거하거나 1.5~2배속한다.
- Backend 터미널의 `AGENT_TRACE` 로그를 5~10초 분할 화면으로 함께 보여준다.

### 2:25~3:25 · Self-Correction

화면:

- risk-review-agent Planning
- 독립 검토 시작
- 대표 Row 변경 전후
- 위험도 시스템 재계산
- 조치 대상 여부 변경

해설:

> 작성 Agent의 결과를 그대로 확정하지 않습니다. 별도 Review Agent가 사고이력 누락과 MSDS 모순을 검토합니다. 수정된 빈도와 강도 후보를 시스템 코드가 다시 계산하고, 위험도 9 이상만 조치계획 대상으로 전달합니다.

강조 예시:

```text
평가 027
검토 전: 빈도 2 × 강도 4 = 8
문제: 최근 감지기 경보 이력이 빈도 근거에 누락
수정 후: 빈도 3 × 강도 4 = 12
조치 대상: 아니오 → 예
```

### 3:25~4:20 · 최종 결과 확인

화면:

- `#3 위험성평가`
- 판단/빈도/강도 근거
- `#4 조치계획서`
- 검토 및 보완 내역
- 결과 Excel 다운로드

해설:

> 모든 결과는 입력 평가 조합과 연결되며 판단, 빈도, 강도 근거를 포함합니다. 시스템이 계산한 위험도 9 이상 항목에만 조치계획이 생성됩니다.

### 4:20~4:50 · 성과 요약

표시할 수치:

- 입력 평가 조합 수
- 생성된 위험성평가 Row 수
- 입력 조합 일치 건수
- 근거 포함 건수
- 독립 검토 수정/유지 건수
- 위험도 9 이상 건수
- 조치계획 연결 건수

해설:

> 이 수치는 AI 전문판단 정확도 100%를 의미하지 않습니다. 입력 보존, 형식, 근거 존재, 위험도 계산과 결과 연결을 시스템이 검증한 수치입니다.

### 4:50~5:00 · 마무리

> AI는 위험 시나리오와 후보값을 생성하고, 시스템은 입력 범위와 계산을 통제하며, 최종 안전 판단은 담당자가 수행합니다.

## 촬영 전 체크리스트

- [ ] Backend 실행 모드가 DeepAgent인지 확인
- [ ] KOSHA 조회 성공 또는 fallback 사유가 로그에 표시되는지 확인
- [ ] 각 Agent의 Planning 실행 현황이 한국어 5/5 완료로 보이는지 확인
- [ ] Tool 호출 목적·입력·결과가 보이는지 확인
- [ ] Self-Correction 대표 Row가 최소 1건 있는지 확인
- [ ] Backend 터미널에서 `AGENT_TRACE`가 보이는지 확인
- [ ] 결과 Excel이 다운로드되는지 확인
- [ ] 전체 영상이 4분 50초 안쪽인지 확인
- [ ] 파일명과 용량 제한을 확인

## Backend 구조화 콘솔 로그 확인 방법

로컬 서버를 실행한 터미널에서 다음 형태를 확인한다.

```text
AGENT_TRACE {"request_id":"...","agent_id":"risk-review-agent","kind":"self-correction",...}
```

Docker 실행 시에는 API 서비스 로그를 따라간다. 실제 서비스 이름은 `docker compose ps`로 확인한 후 아래처럼 실행한다.

```bash
docker compose logs -f <API서비스명>
```

영상에는 전체 콘솔을 오래 보여주지 말고 Planning, Tool, Self-Correction JSON이 나타나는 구간만 5~10초 보여준다.
