# 작업 설계: Azure OpenAI 출력 형식 안정화

## 배경

실제 Azure OpenAI 호출에서 `Azure OpenAI 위험성평가 결과 형식이 올바르지 않습니다: 'risk_rows'` 오류가 발생했다.
이는 LLM 응답이 JSON은 맞지만 최상위 키가 기대한 `risk_rows`가 아닌 형태로 왔다는 뜻이다.

## 수정 범위

1. LLM 프롬프트에 반드시 필요한 최상위 JSON 구조 예시를 명확히 추가한다.
2. LLM 응답이 아래 형태로 와도 복구한다.
   - `{"risk_rows": [...]}`
   - `[...]`
   - `{"rows": [...]}`
   - `{"data": [...]}`
   - `{"action_rows": [...]}`
3. 복구가 불가능할 때는 실제 반환된 최상위 키를 오류 메시지에 표시한다.
4. Agent 로그 문구가 고정 문자열인지/동적 문자열인지 설명 가능한 상태를 유지한다.

## 비수정 범위

- 위험도 계산 기준은 변경하지 않는다.
- Excel 구조는 변경하지 않는다.
- DB 스키마는 변경하지 않는다.
- LLM Provider는 Azure OpenAI를 유지한다.

## 성공 기준

- `risk_rows` 키가 없는 LLM JSON도 가능한 경우 복구된다.
- 복구 불가능 시 오류 메시지에 응답 키가 표시된다.
- demo Agent 흐름과 샘플 3종 테스트는 계속 통과한다.
